Pre-Presentacion
Installar http-serve (Servidor local para angular transpilado)
npm install http-server -g  (-g para que sea global a todos los usuarios)

Presentacion
1-Transpilar el proyecto usando el patron aot(Aparecera en la carpera dist)
  ng build --configuration production --aot

2-Coopilar el projecto java usando maven
  A- Menu maven(lateral derecho)
  B- Nombre del projecto 
  C- Lifecycle
  D- Package
Esto coopilara el proyecto en la carpeta target en un .jar

2- Copiar la carte dist a otra ubicacion(El escritorio por ejemplo), para demostrar que no hay truco y que se ejecutan los archivos transpilados.

3- Abrir VisualStudio en la ubicacion del dist, desde la terminal cd a la ubicacion y ejecutar:
 code. (Punto incluido)

4- Ejecutar el servidor http con el siguiente comando:
 http-server -p 9000 -cors -o material

4.5-Ejecutar java --jar connectDB-0.0.2-SNAPSHOTv2.jar en la carpeta que contenga el backend en formato .jar

5- Se abrira automaticamente el Login (User: Admin, pass= Admin123)

6- Ir a la pesta√±a de usuarios y si se cargaron usuarios de primera la conexion con la base de datos esta establecida
  A- En caso de opuesto comprobar si el usuario en el archivo application.properties (username) sea el usuario que tiene la conexion a la base de datos en el dbeaver(boton derecho en local host y comprobar los detalles de la conexion)

7- Probar las funcionalidades
  A-Usando el boton agregar se realizara el alta del usuario
  A-1. Al momento de agregar puede ocurrir un error en donde no se visualiza el nuevo usuario agragado, esto es una peculiaridad conocida que se encuentra en proceso de ser solucionada 
  A-2. Simplemente realizar cualquier otra consulta o recargar la pagina para que se actualice la lista correctamente

8- Para cerrar las aplicaciones simplemente realizar ctrl+c en ambas terminales y listo
